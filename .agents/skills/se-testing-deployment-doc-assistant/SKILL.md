# SE Testing Deployment Doc Assistant

将测试与部署活动形成“可复现、可验证、可交付”的课程文档。

## 适用模板

春季优先：
- `模版库-供claudeCode调用/提交文件模版/春季/测试报告.md`
- `模版库-供claudeCode调用/提交文件模版/春季/用户手册.md`

夏季优先：
- `模版库-供claudeCode调用/提交文件模版/夏季/用户使用说明书.md`
- `模版库-供claudeCode调用/提交文件模版/夏季/项目总结.md`（测试与部署复盘章节）

## 输入

- 测试计划与测试用例
- 自动化测试输出/手工测试记录
- 缺陷清单（状态、严重级别、修复版本）
- 部署环境、部署脚本、运行截图

## 输出

- 测试范围、方法、结果统计
- 关键缺陷闭环情况
- 部署步骤、回滚策略、运行验证
- 用户操作说明与常见问题
- 已知限制与后续优化

## 流程

### 1. 测试设计与执行对齐

- 用例来源（需求编号）
- 覆盖范围（功能/性能/兼容/安全）
- 实际执行结果与偏差

### 2. 缺陷闭环分析

每个高优先级缺陷记录：
- 复现条件
- 根因分析
- 修复方式
- 回归结果

### 3. 部署文档化

最少包含：
- 环境要求
- 部署命令与参数
- 配置项说明
- 健康检查
- 回滚步骤

### 4. 用户手册整理

- 角色与权限
- 关键业务流程操作
- 错误提示与处理建议
- 常见问题（FAQ）

## 质量检查

- 所有测试结论是否有执行证据
- 已知问题是否与项目总结一致
- 部署步骤是否可被他人复现
- 用户手册是否覆盖核心场景

## 输出风格

- 事实优先，避免“基本完成”这类模糊表达
- 统计指标尽量量化（通过率、缺陷密度、部署耗时）
- 对未解决问题给出风险等级与处置计划
