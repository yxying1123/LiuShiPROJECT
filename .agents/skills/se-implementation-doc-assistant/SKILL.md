# SE Implementation Doc Assistant

聚焦实现阶段，把“做了什么、怎么做的、做到什么程度”写成可核验文档。

## 适用模板

春季优先：
- `模版库-供claudeCode调用/提交文件模版/春季/软件开发计划书.md`

夏季优先：
- `模版库-供claudeCode调用/提交文件模版/夏季/项目管理文档.md`
- `模版库-供claudeCode调用/提交文件模版/夏季/人员分工权重.md`

## 输入证据

- Git 提交记录、分支信息、里程碑
- 模块完成情况与接口联调状态
- 缺陷列表与修复状态
- 任务管理平台（如云效）中的需求/任务/缺陷

## 输出内容

- 实施计划与实际执行对比
- 每日/每迭代进展记录
- 人员分工、贡献与权重说明
- 风险与阻塞项（含应对措施）
- 阶段交付物清单

## 核心流程

### 1. 计划-执行对齐

生成对照表：
- 计划任务
- 实际完成状态
- 偏差原因
- 修正措施

### 2. 实现证据归档

按模块归档：
- 模块目标
- 对应代码路径
- 关键提交
- 完成度与已知问题

### 3. 协作过程记录

- 任务拆分原则
- 角色职责与交接
- 评审与同步机制

### 4. 风险复盘

- 技术风险
- 进度风险
- 依赖风险
- 质量风险

## 写作规范

- 全部结论必须对应可追溯证据
- 对“未完成”内容明确标注剩余工作
- 对“临时方案”标注后续重构计划
- 避免只写流水账，突出决策和结果

## 验收标准

- 教师可根据文档反推开发过程
- 关键里程碑具有真实时间线支撑
- 人员贡献说明透明、可解释
- 与测试和总结文档前后一致
