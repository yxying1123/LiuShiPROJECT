# SE Architecture Doc Assistant

把需求规格转成可实现的系统设计文档，覆盖概要设计与详细设计。

## 适用模板

优先使用春季模板：
- `模版库-供claudeCode调用/提交文件模版/春季/软件概要设计说明书.md`
- `模版库-供claudeCode调用/提交文件模版/春季/软件详细设计说明书.md`

若为夏季项目：
- 将架构设计内容映射到“项目管理文档”的技术方案与迭代说明
- 在“项目总结”中给出架构取舍与重构复盘

## 输入

- 已确认需求文档（功能 + 非功能）
- 技术栈与部署环境
- 当前代码结构（若已有）
- 关键业务流程

## 输出结构

### 概要设计层

- 系统总体架构与分层
- 核心模块职责
- 外部接口与交互关系
- 关键数据结构与数据库设计要点
- 运行与部署约束

### 详细设计层

- 模块内部设计（类/函数/流程）
- 接口定义（参数、返回、错误码）
- 核心算法与状态流转
- 异常处理与日志策略
- 可测试性设计点

## 生成流程

### 1. 架构决策记录

每个关键决策应包含：
- 决策项
- 备选方案
- 选择理由
- 风险与缓解

### 2. 模块与接口拆分

输出模块表：
- 模块名
- 职责边界
- 输入输出
- 依赖关系
- 对应需求编号

### 3. 数据设计

- 核心实体与关系
- 读写路径
- 一致性策略
- 索引与容量预估（可估算）

### 4. 流程设计

- 关键业务时序
- 失败回滚路径
- 并发冲突处理

## 质量门禁

- 架构是否能覆盖全部核心需求
- 模块边界是否清晰，是否存在职责耦合
- 接口是否满足实现与测试直接使用
- 设计内容是否与当前代码事实一致

## 强制约束

- 不编造不存在的组件和中间件
- 不输出与实际技术栈冲突的方案
- 所有图示描述必须可由文字验证
