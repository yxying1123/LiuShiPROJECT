# cluster单细胞数据分析系统用户手册（扩展版）

## 1 软件简介

cluster 单细胞数据分析系统是一套面向流式细胞术、质谱流式以及多样本单细胞表型数据研究场景构建的可视化分析平台，系统采用前后端分离架构，由前端展示层 `cluster` 与后端计算服务 `cluster-End` 共同组成，面向科研教学、课题分析、方法验证和结果复现等多种使用环境，提供从原始文件导入、补偿矩阵整合、二维散点展示、UMAP 降维分析、细胞群筛选、聚类计算、热图展示到结果保存与导出的完整工作链路。与传统需要在多个软件之间反复切换、依赖脚本手工拼接分析步骤的做法相比，本系统把数据准备、分析配置、结果查看和导出归并到统一界面中，既降低了使用门槛，也提升了分析流程的一致性、可追踪性和可复用性，使用户能够将更多注意力集中在细胞群体差异、生物学意义判断和研究结论提炼上，而不是耗费在重复性的数据整理与图形加工工作中。

从系统定位来看，该平台并不仅仅是一个简单的绘图界面，而是一个围绕“单细胞数据探索式分析”构建的轻量级专业工作台。前端以 React 组件化方式组织页面与交互逻辑，能够将文件列表、散点分析、热图聚类和结果页整合为连续的业务流；后端基于 FastAPI 构建高效服务接口，负责承担文件存储、CSV 元数据读取、FCS 补偿与 arcsinh 转换、UMAP 降维、Phenograph 聚类、层次聚类树计算以及 ROE 细胞比例矩阵生成等核心计算任务。这样的架构使系统兼具现代 Web 产品的操作便利性与科研分析工具的算法能力，既适合在课程设计和实验教学中用于演示分析思路，也适合在实验室内部数据预处理、初步探索与图形输出阶段作为常态化工具使用。

在功能设计上，系统强调“面向过程的连续分析体验”。用户首先在文件列表页面完成原始数据导入，既可以直接上传 CSV/TXT 表格文件，也可以上传一个补偿矩阵 CSV 与多个 FCS 文件，由系统自动完成补偿计算、通道转换和结果落库；随后用户在数据分析页面基于已上传文件选择分析来源、数值维度和采样规模，系统将自动完成二维绘图或高维降维，并生成可交互散点图；在散点图界面中，用户可以通过框选方式逐步缩小目标细胞群，对筛选后的数据执行再次降维、聚类分析、自定义分组以及热图计算；当获得满意结果后，还可以把降维结果或聚类结果保存到分析结果页面，以便后续比对、展示和导出。这种“导入数据即进入分析上下文”的设计，能够很好地贴合单细胞分析中从全局探索到局部聚焦、从初步观察到结构化输出的实际工作模式。

在算法与数据处理层面，系统针对单细胞数据的典型特点进行了针对性适配。对于 FCS 数据，系统使用补偿矩阵对信号进行校正，并对检测通道执行以 COFACTOR 为参数的 arcsinh 转换，满足流式与质谱流式数据常见的预处理习惯；对于降维任务，系统通过选择指定数值列并进行标准化处理后调用 UMAP 算法，尽可能保留高维数据中的局部邻域结构，使散点图不仅具备可视化意义，也具有细胞群体分布探索价值；对于聚类任务，系统使用 Phenograph 计算社区结构，并在此基础上构建按群体均值汇总后的热图矩阵以及行列聚类树，帮助用户快速识别不同聚类之间的表达差异、特征相关性与群体组织关系；对于 ROE 细胞比例热图，系统进一步基于聚类标签与来源标签构建列联表，并输出 log2(Ro/E) 结果，使用户能够从比例富集或相对缺失的角度观察不同样本来源中的群体分布变化。

在可视化与交互层面，本系统强调分析界面与结果解释之间的直接联动。散点图不仅仅用于展示点位，还承担了筛选、回退、重做、着色、图例过滤、自定义分组、预览生成和导出等多重角色；热图页面也并非单纯的静态图像容器，而是将热图树、散点结果、聚类表和散点表整合在统一视图之中，支持从多个角度核对结果是否可信、是否具备解释价值。尤其是在图例层面，系统支持默认模式与自定义模式之间切换，用户既可以直接按来源或聚类类别进行筛选，也可以自行组合多个来源或多个聚类形成“自定义组”，并赋予新的名称和颜色，用于观察特定实验批次、特定样本集合或者特定细胞群子集之间的关系。这种面向分析者思维方式的设计，使系统从“数据展示工具”进一步提升为“交互式研究辅助平台”。

从应用价值来看，cluster 单细胞数据分析系统具有较强的综合适用性。一方面，它适合教学与课程项目场景，能够把文件导入、降维、筛选、聚类、热图、导出等步骤以较为直观的界面方式展现出来，有利于学生理解单细胞分析链路中的关键处理节点；另一方面，它也适合实验室日常科研使用，尤其适用于免疫分群、表型探索、多样本对比、预实验验证以及图形整理等阶段，能够帮助研究人员快速完成从原始数据到初步结论图景的转化。对于需要在多批次样本之间比较细胞群差异、在复杂散点结构中定位目标亚群、或在汇报材料中快速输出高质量结果图和结果表的用户而言，该系统能够显著提升工作效率、减少重复劳动，并提高分析过程的可记录性与结果表达的规范性。

总体而言，本系统将文件管理、单细胞降维分析、交互筛选、群体聚类、表达热图、细胞比例热图和结果归档整合为一个有机整体，在保证操作门槛适中的同时保留了较强的专业分析能力。它既体现了现代 Web 技术在科研软件中的工程化落地能力，也体现了单细胞分析流程“从数据到结构、从结构到解释”的方法逻辑，能够为课程实践、科研探索和项目演示提供稳定、清晰且具有扩展空间的软件支撑。

## 2 硬件环境

### 2.1 硬件配置要求

本系统面向单机部署与局域网内部部署场景设计，既可以在普通开发电脑上完成基础分析，也能够在配置较高的工作站或实验室服务器上处理更大规模的数据集。考虑到单细胞数据通常具有维度多、记录数大、计算链路包含标准化、降维、聚类和图形渲染等多个阶段，硬件配置将直接影响文件读取速度、UMAP 计算时间、热图生成速度以及浏览器端交互流畅性，因此建议根据实际数据体量选择合适的部署方案。

| 配置项 | 最低配置 | 推荐配置 | 高性能建议 |
|------|------|------|------|
| 处理器 | 4 核 CPU，主频 2.5GHz 以上 | 8 核 CPU，主频 3.0GHz 以上 | 12 核及以上高性能处理器 |
| 内存 | 8GB | 16GB | 32GB 及以上 |
| 存储 | 20GB 可用空间 | 100GB SSD | NVMe SSD，便于批量文件读写 |
| 显示设备 | 1366×768 分辨率 | 1920×1080 分辨率 | 2K/4K 显示器，便于查看热图细节 |
| 网络环境 | 单机本地访问 | 千兆局域网 | 局域网服务器或实验室内网部署 |

在最低配置条件下，系统可以完成中小规模 CSV 文件上传、基础二维散点分析、有限点数的降维和聚类任务，但当数据文件数量较多、FCS 文件体积较大、筛选后仍保留大量细胞点或同时生成多张预览图时，浏览器渲染与后端计算的响应时间会明显增加。推荐配置能够满足课程演示、常规实验分析和多文件联合对比场景，尤其适合需要频繁执行聚类分析、热图查看和结果导出的用户。若系统部署于科研工作站或服务器环境中，并配置更高核心数 CPU 与更大内存，则更适合承担大样本、多批次和长时间连续分析任务，能够显著提升 UMAP、Phenograph 与层次聚类计算效率，同时保证前端界面在大规模点云渲染时的稳定性。

### 2.2 软件运行环境

系统前端建议运行在支持现代 JavaScript 标准的主流浏览器中，如 Chrome、Edge 或 Firefox 的较新版本，以确保 Plotly 绘图、页面动画、对话框交互和本地文件下载功能能够稳定工作。前端开发与构建建议使用 Node.js 18 及以上版本，配合 npm 进行依赖管理和项目构建。后端建议使用 Python 3.9 及以上版本运行 FastAPI 服务，并安装 Pandas、NumPy、scikit-learn、umap-learn、SciPy、FlowKit 等依赖库，以保证文件读取、数值处理、降维聚类和 FCS 转换功能完整可用。对于需要长期运行或局域网共享的部署环境，建议在 Linux、macOS 或 Windows 64 位系统上使用独立 Python 虚拟环境进行安装，以避免不同项目间的依赖冲突影响系统稳定性。

### 2.3 外设与使用建议

如果用户在实验室环境中需要频繁进行图像导出、结果展示和汇报截图，建议搭配高分辨率显示器使用，以便更清晰地观察散点边界、热图标签和聚类结构。当系统承担教学展示任务时，建议使用响应较快的固态硬盘和至少 16GB 内存的设备，以保证课堂现场切换页面、导入文件和生成图形时操作连贯。在批量处理 FCS 文件或进行多轮聚类参数调试时，建议优先关闭其他高占用应用，保留足够内存与 CPU 资源供后端计算模块使用，从而减少等待时间并提高整体分析效率。

## 3 主要功能

### 3.1 文件列表

#### 3.1.1 CSV 文件导入

CSV 文件导入功能主要用于将已经整理好的单细胞表格数据直接纳入系统管理范围。用户进入文件列表页面后，可通过右上方的“CSV文件导入”按钮打开上传窗口，在窗口中选择一个或多个 CSV 或 TXT 文件并提交，系统会立即将文件传输到后端存储目录，并自动刷新文件列表区域，使用户能够在同一页面中看到新文件的名称、大小和最近修改时间。该功能适合已经完成基础清洗、转换或预处理的数据表直接进入分析流程，也适合用于课程演示时快速切换不同案例数据。由于系统会在后续分析阶段自动提取列信息与数值字段，因此用户无需在导入时完成复杂配置，只要保证文件结构规范、列名明确且包含足够的数值型指标，即可在后续页面中直接参与二维散点和降维分析。

#### 3.1.2 FCS 文件导入

FCS 文件导入功能面向更贴近原始实验数据的使用场景，允许用户在系统中直接完成补偿矩阵应用、通道信号转换和 CSV 结果生成。操作时，用户点击“FCS文件导入”按钮，系统会弹出专门的数据整合窗口，并要求用户同时上传一个 CSV 格式的补偿矩阵文件与一个或多个 FCS 数据文件，然后输入 COFACTOR 参数以控制 arcsinh 转换强度；当前端提交请求后，后端会自动识别 CSV 与 FCS 文件类型，对 FCS 数据应用补偿矩阵、对检测通道完成转换，并按照原始 FCS 文件名生成对应 CSV 文件保存到系统文件库中，最终这些新生成的结果文件会出现在文件列表里供后续分析直接使用。该功能显著减少了用户在外部软件中预转换数据的工作量，使系统具备从原始实验文件直接进入分析流程的能力，特别适合流式细胞术与质谱流式数据的统一管理和快速处理。

#### 3.1.3 文件管理

文件管理功能承担系统数据入口与数据资产整理中心的作用。用户在文件列表页面不仅可以看到所有已上传或由系统整合生成的文件，还可以对每个文件执行下载和删除操作，从而实现结果回溯、原始文件留存与无效文件清理的统一管理。当用户点击下载按钮时，系统会直接从后端存储目录返回文件内容并触发浏览器保存；当用户删除文件时，系统会同步移除对应存储项并刷新页面状态，如果当前文件已被分析流程引用且删除后文件集合为空，系统还会自动重置已有分析状态，避免界面残留无效结果。通过这一机制，文件列表页面不仅是“上传入口”，更是系统分析上下文的基础管理层，保证用户在进行后续散点分析和聚类前始终基于清晰、可控的数据集合开展工作。

### 3.2 数据分析

#### 3.2.1 画图分析

画图分析是整个系统的数据探索起点，负责把文件中的高维或二维数值字段快速转化为可交互散点图。用户在数据分析页面发起“新建分析”后，可以选择参与分析的文件、指定分析模式、勾选降维维度或选择二维坐标轴，并根据数据规模设置随机读取行数；当用户确认配置后，系统会读取选定文件的列信息与样本数据，对二维模式直接提取指定坐标字段，对降维模式则将所选维度送入后端执行 UMAP 计算，随后把返回的结果整合为前端散点图并按来源或数值字段着色显示。由于该模块既支持传统二维散点观察，也支持面向多维指标的降维展示，因此能够同时满足“验证两个通道关系”与“探索高维群体结构”两类不同分析诉求，是后续筛选、再次降维和聚类分析的基础入口。

#### 3.2.2 数据筛选

数据筛选功能使用户能够直接在散点图上圈定感兴趣的细胞区域，并将视觉判断即时转化为后续分析对象。用户在图中使用框选工具选中一批数据点后，点击“确定筛选”，系统会把当前完整散点状态存入历史记录，再将被选中的点替换为新的分析主体，从而实现从整体分布向局部亚群逐步缩小范围的探索过程。如果用户发现筛选边界不理想，还可以利用“清除选择”取消当前框选，或通过历史回退与前进机制恢复到上一步或下一步状态。该功能适用于在复杂分布中锁定特定细胞群、剔除明显噪声点或逐层逼近目标亚群的情形，并且能够与后续的降维优化、聚类与热图展示形成紧密联动。

#### 3.2.3 降维分析

降维分析功能用于在已有散点结果或筛选结果基础上重新选择变量并执行新一轮 UMAP 计算，从而帮助用户观察不同维度组合下群体结构是否发生更清晰的分离。用户在散点图页面点击“降维分析”后，可以在弹出的维度选择对话框中重新勾选参与降维的数值字段，系统会将当前点集对应的原始指标与用户新选择的维度一起发送到后端，由后端完成标准化和 UMAP 计算，再把新的二维坐标返回前端并刷新显示。与首次建图不同，该功能更强调“基于当前观察结果的二次重构”，尤其适合在初始降维效果不理想、某些变量可能干扰分群结构、或已经筛选出某个子群后需要继续观察其内部异质性时使用，因此是系统中支持深度探索的重要能力。

#### 3.2.4 保存降维分析

保存降维分析功能用于把当前散点结果固定为一个可重复查看、可独立导出的分析快照。用户在散点图生成完成后点击“保存散点图”，输入具有识别意义的名称，例如样本来源、筛选目标或实验条件说明，系统便会将当前散点数据、保存时间和结果类型写入结果列表中，使该结果在后续会话中仍可作为独立项目查看。这样一来，用户无需担心在继续筛选、继续降维或尝试不同分析策略时丢失当前阶段成果，也可以把不同版本的降维结果分别保存下来用于比较。该功能对课程汇报、实验记录和多轮参数调优尤为重要，因为它让“分析过程中的阶段性成果”能够像最终结果一样被清晰归档。

#### 3.2.5 聚类分析

聚类分析功能是在当前散点点集基础上执行群体划分和表达矩阵构建的核心模块。用户在数据分析页面点击“聚类分析”后，可以输入或调整近邻数 `k`、分辨率 `resolution`、迭代次数 `n_iterations`、随机种子 `seed` 等参数，并可指定需要排除的变量或热图聚合后不再展示的字段；确认后，系统会将当前点集的数值数据提交给后端执行 Phenograph 聚类，并按群体均值生成热图矩阵，同时计算行树与列树结构，最后把群体标签重新回写到散点图中并展示热图结果。该功能不仅完成了“点如何分群”的计算，还进一步建立了“分群之后每个群体在不同指标上的表达轮廓”，因此是连接散点可视化与生物学解释的关键节点。

#### 3.2.6 保存聚类分析

保存聚类分析功能用于把已经生成的聚类热图、群体标签和散点数据整体归档为一个完整结果集。用户在聚类成功后，无论位于当前分析页面还是结果聚类页面，都可以通过“保存结果”输入名称并完成保存，系统会记录该结果的保存时间、点数规模、热图矩阵大小和关联散点数据，使其在后续页面中以独立结果卡片形式展示。这样，用户可以将不同参数下得到的聚类结果分别保留，随后对比各版本的热图结构、聚类数量和散点分布差异，而不必反复重新运行计算。对于需要比较不同参数策略、不同筛选层级或不同样本子集的实验场景而言，这一功能大幅提升了结果管理效率和分析复现能力。

#### 3.2.7 自定义筛选

自定义筛选功能建立在图例过滤基础之上，允许用户不再局限于系统自动生成的来源类别或聚类编号，而是根据研究目的自行构造更贴近业务含义的筛选集合。用户可以在来源筛选或聚类筛选区域切换到“自定义”模式，将多个来源标签或多个聚类标签合并为一个新的自定义组，并为其命名和赋予专属颜色；此后散点图、预览图和热图计算都可以基于这些自定义组进行展示和筛选。这样的设计尤其适合在多批次样本中把同类实验组合同步观察，或把若干相近的聚类亚群视为一个功能群体进行对比分析。相比简单的勾选隐藏，自定义筛选进一步提供了“重组类别结构”的能力，使图例系统从被动显示工具转变为主动分析工具。

#### 3.2.8 自定义配置

自定义配置功能主要体现在分析过程中的显示、着色和预览参数调节上，帮助用户根据当前任务切换不同观察视角。用户可以在散点图界面选择不同颜色模式，如按来源着色、按聚类着色或按数值字段连续着色，也可以调整点大小、切换图例模式、设置热图标准化方式，并在需要时通过预览功能按当前分类一键生成多张散点子图。通过这些配置，系统不仅支持“有没有结果”的判断，更支持“结果是否足够清晰、是否适合展示、是否便于解释”的进一步优化。对于需要制作汇报图、答辩图或论文初稿插图的用户而言，这些可配置项能够显著减少在外部绘图软件中二次加工的成本。

#### 3.2.9 分析流程记录

分析流程记录功能用于保存用户在当前会话中的关键操作轨迹，使分析过程具备可回溯性和上下文连续性。系统会把新建分析、确定筛选、降维分析、聚类分析等重要节点记录到分析流程区域，并在必要时保留对应结果快照，用户既可以直观理解自己当前处于哪一步，也能在需要时回看历史聚类结果而不影响当前散点状态。对于探索性分析来说，这一能力非常重要，因为用户往往会在多轮尝试中逐步确认最佳路径，而不是一次操作就得到最终结果。流程记录把这些“试探式分析步骤”组织为可读、可追踪的过程信息，有助于后续整理实验思路、编写报告和复盘参数设置。

#### 3.2.10 展示表达量热图

表达量热图功能用于在聚类分析完成后，从群体均值层面展示不同聚类在各数值指标上的相对表达模式。用户可以点击“表达量热图”或查看当前热图，系统会根据当前散点数据、图例分组和热图标准化策略计算热图矩阵，并以彩色矩阵结合聚类树的方式呈现各群体与各特征之间的高低表达关系。若当前处于默认聚类结果模式，系统将展示包含行树与列树的完整热图结构；若用户使用了自定义分组模式，系统也可以基于自定义组重新生成简化版热图，从而帮助用户观察组合后的来源组或聚类组是否具有更明显的表达差异。该功能是单细胞结果解释的重要支撑，因为它把“散点上看见的分群”转换为了“指标层面可说明的群体特征”。

#### 3.2.11 展示细胞比例热图

细胞比例热图功能聚焦于不同来源样本中各聚类群体的相对富集情况，是系统中连接细胞组成差异与样本比较的重要统计展示模块。用户在完成聚类之后点击“细胞比例热图”，系统会从当前散点结果中提取聚类标签和来源标签，构建列联表，并计算观测比例与期望比例之比，再以 log2(Ro/E) 的形式生成热图。这样一来，用户不仅能够看到某个聚类是否存在，还能进一步判断它在某类样本中是相对富集、相对减少还是接近期望分布。对于免疫分群比较、病例与对照样本差异分析、实验处理前后细胞组成变化等场景，该功能能够提供比简单计数更有解释力的结果视角。

### 3.3 分析结果

#### 3.3.1 降维分析结果展示

降维分析结果展示页面用于统一保存和查看此前归档的散点分析成果。系统会把用户保存的降维结果以卡片形式展示出来，卡片中包含结果名称、保存时间、数据点数量等信息，用户展开后可以下载完整降维数据表，也可以下载去除坐标列与聚类字段后的精简版表格，以满足继续分析、结果汇报或数据交接等不同使用需求。由于系统会将每一次保存视为一个独立结果对象，因此用户能够在同一页面中保留不同样本组合、不同筛选策略或不同降维变量组合下的多个结果版本，从而方便后续进行横向比较和阶段性总结。

#### 3.3.2 聚类分析结果展示

聚类分析结果展示页面用于集中管理已经保存的聚类结果，是系统中最完整的结果归档与输出区域。每个聚类结果不仅包含保存名称与时间，还能够展开查看散点数据规模、热图矩阵大小和聚类状态，并支持下载完整降维数据、精简数据、热图表格以及相关图片资源；在部分视图中，用户还可以进一步查看聚类热图树、聚类表和散点表，从而对结果进行核验和展示。该页面的价值在于把“计算阶段的临时结果”转化为“具有命名、时间戳和导出能力的正式结果资产”，使整个系统不仅能做分析，也能支撑实验记录、课程提交、阶段汇报和成果留档。

## 4 结语

cluster 单细胞数据分析系统通过统一的文件管理机制、灵活的散点探索流程、可配置的降维与聚类能力、直观的热图展示方式以及完善的结果保存与导出功能，构建出一条覆盖“原始数据导入、交互式分析、结构化展示、结果归档”的完整业务链路。对于希望快速完成单细胞数据初步探索、对不同样本间细胞组成进行对比、或需要在课程与科研场景中输出规范化分析材料的用户而言，该系统能够提供较高的使用效率、较强的结果表达能力以及较清晰的过程记录能力，具有良好的实用价值与扩展潜力。
