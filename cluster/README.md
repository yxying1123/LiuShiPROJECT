# 环境安装指南
## NVM 安装
``` 
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.1/install.sh | bash
```
## Node.js 18 安装
```
nvm install 18
nvm use 18
```
## 启动开发服务器
```
npm run dev
```

# 功能实现
## 前端页面规范：
页面根据窗口适配
颜色、间距、字体必须使用统一设计变量


## 具体功能
1. 上传文件
允许上传单个或多个文件

2. 选择x轴和y轴的变量
通过文件里面的变量进行选择
展示散点图，当是来源是同一个文件时散点图展示同一种颜色；当是不同文件来源时，同一个文件来源的展示同一种颜色；


3. 降维选择
展示散点图
允许重新重复选择框选区域，和展示点，

4. 结果聚类
点击结果聚类，展示对应框选的点的聚类结果，展示热图聚类树

## 开发规范
请为前端页面设计一套 响应式字体规范，遵循以下原则：
	1.	禁止写死字体大小（如 px 固定值），字体必须根据屏幕尺寸和分辨率自适应变化。
	2.	优先使用相对单位，如 rem、em、vw / vh、clamp()，而不是固定像素。
	3.	页面在 小屏幕（笔记本 / 平板）上字体不过大、不拥挤，在 大屏幕（2K / 4K 显示器）上字体不过小、可清晰阅读。
	4.	根字体大小应根据 视口宽度（viewport width）动态调整，并设置合理的最小值和最大值，避免极端情况。
	5.	标题、正文、辅助文字需有 明确的层级比例，但比例关系应随屏幕尺寸整体缩放。
	6.	字体缩放需保证 可读性优先，避免因屏幕变大导致文字“看起来变小”的问题。
	7.	不同分辨率下（如 1366px / 1920px / 2560px / 3840px），整体阅读体验保持一致。
	8.	字体方案需兼容主流浏览器，避免依赖不稳定或实验性特性。
	9.	行高（line-height）需随字体大小自适应，保证段落在大屏和小屏下都不松散、不拥挤。
	10.	最终效果应符合 现代前端响应式设计与可访问性标准。
