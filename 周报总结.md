# Cluster App 项目周报总结

**项目名称**：Cluster App - 流式细胞术/CyTOF 单细胞数据分析平台  
**汇报周期**：2026年2月25日 - 2026年2月26日  
**技术栈**：FastAPI + React + UMAP + Phenograph/Leiden + PyInstaller  
**项目简介**：本项目是一个面向生物信息学研究者的单细胞数据分析工具，用于流式细胞术和CyTOF数据的降维、聚类与可视化分析。

---

## 第一周 (2026年2月25日) - 项目初始化与架构搭建

### 一、项目仓库初始化

本周首要任务是搭建完整的项目骨架。项目采用前后端分离的架构设计，前端负责数据可视化与交互，后端负责数据处理和算法计算。

在项目结构设计上，我将代码划分为两个主要目录：cluster目录存放前端React应用，cluster-End目录存放后端FastAPI服务。这种命名方式保留了原始项目的特征，同时清晰地表达了前后端的职责边界。

项目初期遇到的一个技术决策是关于Git子模块的处理。原始项目使用了子模块来管理依赖，但这会增加协作复杂度和CI/CD配置的困难。经过评估后，我决定移除子模块依赖，改为将相关代码直接纳入主仓库管理。这个改动虽然增加了仓库体积，但显著简化了构建流程，使GitHub Actions能够直接访问所有源代码而无需处理子模块的递归拉取。

### 二、前端工程化体系搭建

前端部分采用Vite作为构建工具，相比传统的Create React App，Vite提供了更快的开发服务器启动速度和热更新响应。项目配置了以下核心技术栈：

React版本选用18.2，配合React Router v6实现单页面应用的路由管理。样式系统方面，我引入了Tailwind CSS作为原子化CSS框架，这使我们能够快速构建响应式界面而不需要编写大量的自定义CSS。为了加快组件开发速度，集成了Radix UI非样式化组件库，包括对话框、标签页、下拉菜单、表格、复选框、滑块等20多个基础组件，这些组件提供了完善的无障碍访问支持。

数据可视化是本项目的技术核心。散点图部分选用了Plotly.js，它支持WebGL渲染，能够流畅展示数万个数据点的交互式散点图，支持框选、缩放、平移等交互操作。热图聚类树部分则使用D3.js进行自定义渲染，因为D3在树形数据结构的可视化方面提供了更灵活的底层API。

此外，还集成了一些提升用户体验的库：Framer Motion用于页面过渡动画，Sonner用于全局消息通知，html-to-image用于图表导出，PapaParse用于CSV文件解析，Axios用于HTTP请求。

### 三、后端算法服务搭建

后端采用FastAPI框架构建，利用其自动API文档生成和异步请求处理能力。核心算法模块封装在service/reduction.py文件中，主要实现以下功能：

首先是数据采样模块。考虑到单细胞数据通常包含数万甚至数十万个细胞，直接处理全部数据会造成前端渲染压力和后端内存占用。因此实现了随机采样功能，用户可以指定每个文件采样多少行数据（lineNum参数），采样过程使用固定的random_state=42保证可复现性。

其次是UMAP降维算法集成。使用umap-learn库将高维数据投影到二维平面，配置了n_neighbors=15、min_dist=0.001、target_weight=0.5等参数。这些参数经过调优，在保持局部结构的同时拉开不同细胞群体的距离。UMAP采用n_jobs=-1启用多核并行计算，加快降维速度。

第三是聚类算法实现。最初评估了原版Phenograph，但在实际测试中发现其性能较低。因此改用KNN图加Leiden算法的组合实现：首先使用scikit-learn的NearestNeighbors构建k近邻图，然后使用igraph和leidenalg库执行Leiden社区发现算法。这种实现方式在聚类质量和计算速度上都有明显提升，同时支持resolution参数调节聚类粒度。

最后是层次聚类树生成。对于热图展示，需要计算行和列的层次聚类顺序。当环境中安装了SciPy时，使用linkage函数计算层次聚类，dendrogram生成树结构，最后将树结构转换为D3可用的嵌套JSON格式。如果未安装SciPy，则退化为平铺树结构。

### 四、跨平台构建体系配置

为了让非技术用户也能方便地使用本工具，我配置了PyInstaller将Python后端和前端静态文件打包成独立的可执行文件。

在build-win.py中，配置了Windows平台的打包参数。主要技术点包括：将前端build目录作为静态文件嵌入到可执行文件中；添加--hidden-import确保pandas、numpy等库的子模块被正确包含；使用--icon配置应用图标；配置--name指定输出文件名。特别处理了Uvicorn的日志配置，确保在打包环境中stdout和stderr不为None，避免isatty相关的AttributeError。

在build-macos.py中，实现了对macOS ARM64架构的支持。通过接收命令行参数区分不同架构，配置对应的target-arch。移除了对Intel架构的支持，因为当前开发环境以Apple Silicon为主，且可以兼容运行。

GitHub Actions工作流配置了多平台并行构建。当推送版本标签或手动触发时，工作流会分别在windows-latest和macos-latest运行器上执行构建任务。每个任务首先安装Node.js依赖并构建前端，然后安装Python依赖，将前端build目录复制到后端static目录，最后执行平台特定的打包脚本。构建产物作为Artifact上传，最终在release任务中合并并发布到GitHub Release页面。

### 五、本周工作总结

本周完成了项目从零到基础可运行的全部基础设施工作。代码仓库结构清晰，前后端技术选型合理，CI/CD流程自动化程度高。为后续的功能开发奠定了坚实的技术基础。

## 第二周 (2026年2月26日) - 核心功能开发与代码重构

### 一、数据上传与文件管理功能

本周首先实现了文件管理模块，这是整个数据分析流程的起点。

文件上传功能支持多种上传方式：用户可以直接拖拽文件到上传区域，也可以通过点击选择文件。支持多文件同时上传，前端会显示每个文件的上传进度和状态。上传的文件暂存于后端storage目录，后端会对文件进行基础校验，检查文件扩展名是否为允许的.csv、.txt或.fcs格式。

文件列表页面展示所有已上传文件的基本信息，包括文件名、文件大小和最后修改时间。列表按修改时间倒序排列，最新的文件显示在最前面。每个文件条目提供下载和删除操作，删除前会弹出确认对话框防止误操作。

对于FCS格式（流式细胞术标准数据格式）的文件，实现了特殊的预处理流程。用户需要同时上传补偿矩阵CSV文件和FCS数据文件，后端使用flowkit库读取FCS文件，应用补偿矩阵校正荧光信号溢出，然后对数据进行arcsinh转换（使用用户指定的cofactor参数），最后输出为CSV格式供后续分析使用。

### 二、散点图可视化与交互功能

散点图页面是用户最频繁使用的功能模块，实现了完整的单细胞数据可视化与交互分析流程。

页面加载后，用户首先需要选择X轴和Y轴变量。后端会读取上传文件的列信息，识别数值类型的列供用户选择。选择完成后，前端向后端发送请求获取数据，后端根据用户指定的采样行数（lineNum）进行随机采样，返回降维后的二维坐标和原始特征列。

散点图渲染使用Plotly.js的scattergl模式，这种模式利用WebGL在GPU上渲染，能够流畅处理数万个数据点。每个点的颜色根据样本来源自动分配，同一文件的数据点使用相同颜色，便于区分不同样本。

核心的交互功能是框选降维。用户可以在散点图上用鼠标框选一个区域，页面会显示框选区域内的数据点数量。确认选择后，前端将选中点的原始高维特征数据发送到后端的select接口，后端对这些选中的子集重新执行UMAP降维。这个流程模拟了生物信息学分析中的"gate"操作，允许用户在低维可视化中识别感兴趣的细胞群体，然后在降维后的空间中进一步观察其结构。

框选操作支持多次迭代。用户可以重复执行框选和降维，逐步缩小到感兴趣的细胞亚群。每次降维后，散点图会更新显示新的坐标，同时保留原始数据的上下文信息。

### 三、聚类分析与热图可视化

当用户确定了感兴趣的细胞群体后，可以执行聚类分析。聚类功能基于前端的框选数据，将选中点的原始特征发送到后端的heatmap/cluster-tree/points接口。

后端首先对数据执行Phenograph/Leiden聚类，将细胞划分为不同的cluster。默认使用k=50构建KNN图，这个参数可以根据数据密度调节。聚类完成后，按cluster计算每个特征（标记物）的平均表达水平，生成一个cluster-by-marker的均值矩阵。

为了消除不同标记物量纲的影响，对矩阵进行列标准化（z-score标准化），使每个标记物的表达值均值为0、标准差为1。标准化后的数据更适合热图展示，能够突出显示哪些cluster高表达哪些标记物。

层次聚类树计算是热图可视化的关键步骤。对行（cluster）和列（marker）分别执行层次聚类，使用complete连接方式和euclidean距离度量。聚类结果转换为嵌套的树形JSON结构，包含每个节点的名称、距离和子节点信息。这个树结构直接传递给D3.js进行渲染，绘制出带有分支长度的聚类树。

热图页面采用标签页布局，包含热图展示、散点图聚类视图、聚类数据表、散点数据表等多个面板。热图展示面板同时显示行聚类树、列聚类树和热图主体，用户可以通过颜色直观判断不同cluster的标记物表达模式。散点图聚类视图将聚类结果映射回二维散点图，每个cluster使用不同颜色，便于观察聚类在空间上的分布。

### 四、代码重构与工程优化

在功能开发的同时，我对代码进行了大规模重构，以提升可维护性和可读性。

HeatmapPage原始代码约1570行，包含了热图渲染、散点图渲染、数据表、预览图、图例控制等众多逻辑，导致文件臃肿难以维护。重构过程中，我将页面拆分为多个职责单一的组件：热力图展示抽取为HeatmapTab组件，散点图展示抽取为ScatterTab组件，聚类数据表抽取为ClusterTableTab组件，散点数据表抽取为ScatterTableTab组件，图例控制面板抽取为ScatterLegendPanel组件，预览图库抽取为PreviewGallery组件。

同时，将页面逻辑抽象为自定义Hooks。useHeatmapLayout Hook管理热图容器的尺寸计算和窗口大小监听；useScatterPanelLayout Hook处理散点图面板的高度自适应；useFileOperations Hook封装了文件列表的获取、删除、下载、上传等操作；useDataIntegration Hook处理CSV和FCS文件的验证与整合上传逻辑。

工具函数也按照功能分类整理到独立的模块中。exportUtils模块提供CSV导出相关的函数，包括CSV值转义、CSV内容构建、文件下载触发、热图数据导出等；dataHelpers模块提供数据处理的辅助函数，包括数值归一化、范围限制、列选项生成、数据点过滤等。

重构后的HeatmapPage代码量减少到约280行，降幅达82%。文件现在主要负责组件组合和状态管理，具体的渲染逻辑和副作用处理都委托给子组件和Hooks。这种架构使得每个模块的职责清晰，便于后续的功能扩展和问题排查。

FileListPage也进行了类似的重构，从412行减少到185行，降幅55%。抽取出FileCard组件负责单个文件的展示，FileListHeader组件负责列表头部，EmptyState组件负责空状态提示。业务逻辑抽取为useFileOperations Hook。

### 五、Bug修复与问题排查

本周修复了多个构建和运行过程中的问题。

静态文件服务路由优先级问题：FastAPI默认的路由注册顺序导致根路径/返回的是后端API的Hello World响应，而不是前端的index.html。通过调整路由注册顺序，确保静态文件挂载在其他路由之前，使根路径能够正确返回前端页面。

PyInstaller打包资源缺失问题：flowkit库依赖Gating-ML.v2.0.xsd模式文件进行FCS数据解析，但PyInstaller默认不会自动收集这些资源文件。通过在打包配置中添加--collect-all flowkit参数，确保flowkit的所有资源文件都被打包到可执行文件中。

Windows平台Uvicorn日志错误：打包后的应用在启动时会抛出AttributeError，原因是Uvicorn尝试检查stdout的isatty属性，但在PyInstaller环境中stdout可能被重定向为None。通过调整Uvicorn的日志配置，确保stdout和stderr对象不为None，避免了这个错误。

前端构建输出目录不一致问题：最初CI配置中前端构建输出到dist目录，但Vite配置实际输出到build目录。修复了CI脚本中的路径引用，确保前后端构建产物正确合并。

## 技术实现细节补充

### UMAP降维参数说明

项目中使用的UMAP参数经过生物信息学场景调优：n_neighbors=15控制局部与全局结构的平衡，较小的值保留更多局部结构；min_dist=0.001控制嵌入点的紧密程度，较小的值使同类细胞聚集更紧密；target_weight=0.5平衡了降维时的吸引力和排斥力；low_memory=False在内存充足时启用更精确的算法变体。这些参数对于单细胞数据的降维效果较好，能够清晰分离不同细胞群体。

### Phenograph聚类算法改进

原版Phenograph使用louvain算法进行社区发现，但louvain存在分辨率限制和收敛速度问题。本项目改用Leiden算法，它是对louvain的改进，具有更好的社区划分质量和更快的收敛速度。算法实现中，首先使用ball_tree算法构建KNN图，然后将邻接矩阵转换为igraph图对象，最后调用leidenalg.find_partition执行聚类。支持通过resolution参数调节聚类粒度，较大的值产生更多小cluster，较小的值产生更少大cluster。

### 层次聚类树的D3渲染

后端生成的层次聚类树是嵌套的JSON结构，每个节点包含name、distance和children字段。前端使用D3的hierarchy布局计算树的节点位置，然后使用path绘制贝塞尔曲线连接父子节点。为了美观，节点使用圆形标记，文本标签根据节点深度使用不同颜色区分。整棵树支持SVG缩放和平移，便于查看大规模聚类树的细节。
